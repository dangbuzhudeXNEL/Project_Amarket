## 关键事件

### 1. M2-e NewsAnalysis 编排服务
把前面 session 写好的 Classifier (M2-c) + Scorer (M2-d) + AIProvider (M2-g) 串成 pipeline：

```
NewsItem
  → NewsClassifier.classify()        [纯规则，永远跑]
  → AIProvider.analyze_news()        [FallbackChain: Brainmaster → Anthropic → DeepSeek]
  → 全 AI 失败 → SimpleRuleScorer.score()  [兜底]
  → NewsAnalysis 写表（按 processed_by 区分 AI / 规则）
```

**核心实现**：`src/amarket/services/news/analysis.py` (NewsAnalysisService + AnalysisBatchResult)
- async batch 支持（asyncio.Semaphore 限制 AI 并发）
- 幂等：`(news_id, processed_by)` upsert
- 单条失败不拖垮 batch
- 规则路径自动注入 `_CATEGORY_DEFAULT_ACTION` 映射（如 RISK_EVENT → AVOID）
- 10 单测含 fake AI provider 三态（success/fail/disabled）

### 2. M2-f AlertService（P0-P3 决策）
严格按 Spec v3 §8.7 实现：

| 等级 | 触发 |
|------|------|
| P0 | (RISK_EVENT 或 MACRO_POLICY) ∧ imp ≥ 5 ∧ urg ≥ 5 |
| P1 | imp ≥ 4 ∧ urg ≥ 4 |
| P2 | imp ≥ 3 |
| P3 | 其他（仅入库 news_analysis，不写 alerts） |

**核心实现**：`src/amarket/services/news/alert.py`
- 纯函数 `evaluate_alert_level(analysis)` 可单独测
- 幂等 by `(news_id, level)`
- 17 单测含 parametrize 决策表全 8 组 + 批处理

# A 股实时新闻分析与行情看板需求文档

版本：v1.0  
日期：2026-06-17  
产品目标：基于同花顺、东方财富、雅虎财经等新闻与行情信息源，构建一个面向二级市场分析师和交易团队的新闻推送后台程序与前端新闻看板。

---

## 1. 分析前提

作为二级市场分析师，新闻不是孤立阅读对象，而是行情变化的解释变量和交易情绪的触发器。系统需要解决三个问题：

1. 今天发生了什么重要新闻？
2. 这些新闻可能影响哪些指数、板块、个股或风格？
3. 当前行情变化是被新闻驱动，还是由技术面、资金面或情绪面推动？

系统应从实时新闻、指数行情、板块表现、个股异动、海外市场和宏观事件中提取交易相关信号，并形成日报、盘中提醒和前端看板。

---

## 2. 分析师会关注哪些新闻

### 2.1 A 股直接相关新闻

这类新闻优先级最高，因为它们可能直接影响开盘、盘中板块轮动和个股异动。

关注内容：

1. 监管政策：证监会、交易所、金融监管总局、央行、财政部发声。
2. 产业政策：AI、半导体、新能源、地产、消费、医药、军工、低空经济等。
3. 上市公司公告：业绩预告、定增、并购重组、减持、回购、重大合同、停复牌。
4. 板块异动新闻：涨停潮、跌停潮、ETF 异动、期货联动。
5. 资金面新闻：北向资金、ETF 申赎、两融余额、央行公开市场操作。
6. 风险新闻：处罚、立案调查、财务造假、债务违约、退市风险。

### 2.2 海外市场映射新闻

海外市场会影响 A 股开盘风险偏好和相关产业链。

关注内容：

1. 美股三大指数、纳斯达克、费城半导体指数。
2. 大型科技股和 AI 链：英伟达、微软、苹果、谷歌、Meta、AMD、台积电。
3. 中概股：阿里、京东、拼多多、百度、网易、蔚来、理想、小鹏。
4. 港股和 H 股：恒生指数、恒生科技指数、中资金融、互联网、创新药。
5. 日韩市场：日经 225、KOSPI、半导体、存储、面板、电池、汽车链。

### 2.3 宏观与大类资产新闻

这些新闻决定市场风格偏成长、价值、防御还是避险。

关注内容：

1. 美联储、美国 CPI、PCE、非农、PMI、初请失业金。
2. 美债收益率、美元指数、人民币汇率。
3. 原油、黄金、铜、铁矿石、煤炭、天然气。
4. 国内社融、PMI、CPI、PPI、财政政策、地产政策。
5. 地缘政治、制裁、战争、航运、能源供应冲击。

### 2.4 突发事件与黑天鹅

突发事件需要即时推送，不等待日报生成。

关注内容：

1. 战争、制裁、恐怖袭击、重大外交冲突。
2. 金融机构风险、银行流动性风险、债务违约。
3. 重大自然灾害、公共安全事故。
4. 交易系统故障、监管紧急发声。
5. 影响核心产业链的供应中断。

## 3. 关键时间点

### 3.1 固定日报时间点

| 时间点 | 建议时间 | 核心目的 | 输出内容 |
|---|---:|---|---|
| 盘前 | 08:00-08:45 | 判断开盘情绪和当日主线 | 隔夜海外、国内政策、公告、风险事件、今日关注板块 |
| 早盘跟踪 | 09:25-10:15 | 捕捉开盘异动和新闻验证 | 高开低开原因、板块强弱、首批异动个股 |
| 午间 | 11:30-12:20 | 判断上午行情是否延续 | 上午复盘、午后关注、资金流和板块轮动 |
| 尾盘 | 14:30-15:00 | 捕捉尾盘资金和隔夜风险 | 尾盘拉升/跳水、资金抢筹或避险 |
| 收盘后 | 15:15-16:30 | 形成当日正式日报 | 全天复盘、新闻归因、明日关注、策略提示 |
| 晚间 | 20:00-22:30 | 捕捉公告和海外开盘 | 重要公告、海外风险、美股开盘影响 |

### 3.2 实时告警时间点

以下情况不受固定时间限制，应即时推送：

1. 重大政策发布。
2. 指数 5 分钟内快速拉升或下跌。
3. 重要板块出现涨停潮或跌停潮。
4. 核心个股突发公告。
5. 海外市场或大宗商品剧烈波动。
6. 黑天鹅事件出现。

### 5.1 一级分类

| 一级分类 | 说明 |

| 宏观政策 | 货币、财政、地产、产业、监管 |
| 市场行情 | 指数、板块、成交额、涨跌停、资金面 |
| 公司公告 | 业绩、回购、并购、减持、合同、处罚 |
| 海外映射 | 美股、港股、日韩、中概、海外科技链 |
| 大宗商品 | 原油、黄金、有色、黑色、农产品 |

| 资金流 | 北向、ETF、两融、机构、龙虎榜 |
| 交易提示 | 停复牌、解禁、分红、转债、申购 |

示例：

3. CPO / PCB。
4. 新能源车。
5. 光伏储能。
6. 创新药。
7. 消费。
8. 券商。

10. 军工。
11. 低空经济。

### 5.3 新闻结构化字段

每条新闻入库后应包含：

| 字段 | 说明 |
|---|---|
| news_id | 新闻唯一 ID |
| title | 标题 |
| summary | 摘要 |
| source | 来源 |
| url | 原文链接 |
| published_at | 发布时间 |
| fetched_at | 抓取时间 |

| related_sectors | 影响板块 |

| importance_score | 重要性 1-5 |
| urgency_score | 紧急度 1-5 |

| impact_horizon | 即时、日内、短期、中期 |
| action_hint | 观察、关注、加仓、减仓、规避 |
| ai_reasoning | AI 分析逻辑 |
| duplicate_group_id | 去重事件组 ID |

## 6. 新闻评分模型

### 6.1 重要性评分

重要性评分用于决定新闻是否进入日报、看板或推送。

评分因素：

1. 来源权威度。
2. 是否官方发布。
3. 是否涉及政策或监管。
4. 是否影响多个板块。
5. 是否影响当前热点。
6. 是否涉及持仓或关注标的。
7. 是否已被市场价格验证。

### 6.2 紧急度评分

紧急度用于决定是否即时推送。

高紧急度条件：

1. 黑天鹅。
2. 官方重大政策。
3. 指数或板块快速异动。
4. 个股重大停牌、监管、处罚。
5. 海外市场大幅波动。

### 6.3 情绪方向

情绪分为：

1. 强利多。
2. 利多。
3. 中性。
4. 利空。
5. 强利空。
6. 不确定。

## 7. 后台程序需求

### 7.1 核心模块

后台程序应包含：

1. 数据采集模块。
2. 新闻去重模块。
3. 新闻分类模块。
4. AI 分析模块。
5. 行情联动模块。
6. 推送告警模块。
7. 日报生成模块。
8. 看板 API 模块。
9. 日志与审计模块。
10. 配置管理模块。

### 7.2 数据采集模块

功能：

1. 定时抓取同花顺、东方财富、雅虎财经新闻。
2. 支持增量抓取。
3. 支持失败重试。
4. 支持来源健康检查。
5. 支持备用源切换。

建议频率：

| 时间段 | 频率 |
|---|---|
| 08:00-09:30 | 1 分钟一次 |
| 09:30-11:30 | 1-3 分钟一次 |
| 11:30-13:00 | 3-5 分钟一次 |
| 13:00-15:00 | 1-3 分钟一次 |
| 15:00-16:30 | 3-5 分钟一次 |
| 夜间 | 5-15 分钟一次 |

### 7.3 新闻去重模块

去重逻辑：

1. URL 去重。
2. 标题精确去重。
3. 标题相似度去重。
4. 同事件多源聚合。
5. 同事件更新追加时间线。

### 7.4 AI 分析模块

输入：

1. 新闻标题。
2. 新闻摘要。
3. 来源。
4. 时间。
5. 相关行情。
6. 近期同类新闻。

输出：

1. 分类。
2. 影响板块。
3. 相关标的。
4. 情绪方向。
5. 重要性评分。
6. 日内影响判断。
7. 操作提示。
8. 风险提示。

### 7.5 日报生成模块

日报包括：

1. 今日核心结论。
2. 市场状态。
3. 重要新闻列表。
4. 板块趋势。
5. 个股异动。
6. 海外映射。
7. 风险事件。
8. 明日或午后关注。
9. 操作建议。

### 7.6 推送告警模块

推送渠道：

1. 飞书。
2. 企业微信。
3. 邮件。
4. 站内通知。

推送等级：

| 等级 | 说明 | 动作 |
|---|---|---|
| P0 | 黑天鹅或重大政策 | 即时强提醒 |
| P1 | 重要板块或持仓相关 | 即时推送 |
| P2 | 普通重要新闻 | 汇总推送 |
| P3 | 一般新闻 | 仅入库和看板展示 |

## 11. 验收标准

### 11.1 后台程序

1. 能抓取至少 3 个来源的新闻。
2. 能进行新闻去重和分类。
3. 能生成 AI 分析结果。
4. 能按固定时间生成日报。
5. 能对 P0/P1 新闻即时推送。
6. 能提供前端看板 API。
7. 能记录推送和错误日志。

### 11.2 前端看板

1. 能展示实时新闻流。
2. 能展示板块趋势。
3. 能展示重要告警。
4. 能查看新闻详情和 AI 分析。
5. 能查看日报。
6. 能筛选新闻来源、分类、情绪、板块。

---

## 12. 风险与注意事项

1. 免费网页源可能变化，需要 Adapter 隔离。
2. 抓取频率要控制，避免触发反爬或违反网站规则。
3. 重要新闻必须保留来源链接，避免 AI 编造。
4. AI 分析必须有置信度和风险提示。
5. 操作建议不能替代人工决策。
6. 对外展示时需避免构成个性化投资建议。

---

## 13. MVP 建议

第一阶段只做：

1. 同花顺 + 东方财富 + 雅虎财经新闻采集。
2. 规则分类 + AI 摘要。
3. 三个固定日报：盘前、午间、收盘。
4. P0/P1 告警推送。
5. 前端新闻看板。
6. 板块趋势和新闻热度展示。

暂不做：

1. 自动交易。
2. 复杂策略回测。
3. 完整持仓管理。
4. 多用户权限。
5. 付费数据源深度接入。
